## 4. Принципы дальнейшей работы

1. **Никаких выдуманных функций.** Если функция не найдена в IDA Pro листинге — она не добавляется в код.
2. **Точные размеры.** Все структуры проверяются через `static_assert` на соответствие размерам из бинарника.
3. **Нейтральные имена.** Если назначение поля неизвестно — используется `field_XX` с комментарием о смещении.
4. **Минимальный C++.** Обертки только для удобства, без добавления новой логики.
