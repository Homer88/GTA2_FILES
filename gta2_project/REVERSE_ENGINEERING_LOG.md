# Лог реверс-инжиниринга GTA 2
## Справочник переименований структур, полей и функций

Этот документ фиксирует процесс перевода низкоуровневых данных из IDA Pro (имена вида `field_0xXX`, `FUN_XXXXXX`) в осмысленные имена переменных и функций на языке C.

**Дата обновления:** 2023-10-27
**Версия базы:** GTA 2 (PC)
**Инструменты:** IDA Pro, Ghidra, собственный анализ дампов памяти.

---

## 1. Структура `Car` (Автомобиль)
**Файл:** `include/car.h`  
**Размер:** ~4096 байт (включая вложенные структуры)  
**Описание:** Основная сущность транспорта. Содержит физику, состояние двигателя, двери, пассажиров и визуализацию.

### Поля структуры

| Смещение (Offset) | Старое имя (IDA) | Новое имя (Проект) | Тип данных | Описание / Логика |
|:---|:---|:---|:---|:---|
| **0x00** | `field_0x0` | `Flags` | `u32` | Битовые флаги состояния. Бит 0: активна, Бит 1: повреждена, Бит 2: двигатель заведен. |
| **0x04** | `field_0x4` | `Type` | `CarType` | Enum типа кузова (седан, грузовик, спорт, спецтранспорт). Влияет на физику и спрайт. |
| **0x08** | `field_0x8` | `Position` | `Vector2D` | Координаты центра машины в мире (X, Y). |
| **0x10** | `field_0x10` | `Angle` | `f32` | Угол поворота корпуса в радианах (0.0 = смотрит вверх/север). |
| **0x14** | `field_0x14` | `Velocity` | `Vector2D` | Вектор текущей линейной скорости. Используется для инерции. |
| **0x1C** | `field_0x1c` | `Speed` | `f32` | Скалярная величина скорости (км/ч условно). Для спидометра и ИИ. |
| **0x20** | `field_0x20` | `AngularVelocity` | `f32` | Скорость вращения вокруг оси Z. Зависит от скорости и поворота колес. |
| **0x24** | `field_0x24` | `Health` | `u16` | Прочность кузова. Макс ~1000. При 0 вызывается `Car_Explode()`. |
| **0x26** | `field_0x26` | `ModelID` | `u16` | Индекс спрайта или 3D модели в ресурсах игры. |
| **0x28** | `field_0x28` | `Driver` | `Passenger` | Структура водителя. Содержит указатель на `Ped*` и статус (в машине/садится). |
| **0x38** | `field_0x38` | `Passengers[4]` | `Passenger[4]` | Массив структур пассажиров. Максимум 4 человека. |
| **0x58** | `field_0x58` | `Engine` | `EngineStruct` | Параметры двигателя: обороты (RPM), температура, состояние стартера. |
| **0x88** | `field_0x88` | `DoorLF` | `CarDoor` | Состояние левой передней двери (угол открытия, анимация). |
| **0x98** | `field_0x98` | `DoorRF` | `CarDoor` | Состояние правой передней двери. |
| **0xA8** | `field_0xa8` | `DoorLR` | `CarDoor` | Состояние левой задней двери. |
| **0xB8** | `field_0xb8` | `DoorRR` | `CarDoor` | Состояние правой задней двери. |
| **0xC8** | `field_0xc8` | `BoundingBox` | `Rect2D` | Ориентированный прямоугольник (OBB) для проверки коллизий. |
| **0xD8** | `field_0xd8` | `LastCollision` | `CollisionInfo` | Данные о последнем столкновении: сила, нормаль, объект удара. |
| **0xF0** | `field_0xf0` | `Turret` | `Turret` | Данные турели (если машина военная). Угол, прицел, патроны. |
| **0xF8** | `field_0xf8` | `LightsState` | `u8` | Состояние фар. 0: выкл, 1: вкл, 2: мигают. |
| **0xF9** | `field_0xf9` | `SirenState` | `u8` | Состояние сирены. Активна только у полиции/скорой. |
| **0xFC** | `field_0xfc` | `ColorPrimary` | `u8` | Основной индекс цвета палитры (кузов). |
| **0xFD** | `field_0xfd` | `ColorSecondary` | `u8` | Вторичный индекс цвета (полосы, детали). |

### Функции работы с Car

| Старое имя (IDA) | Новое имя (Проект) | Описание логики |
|:---|:---|:---|
| `FUN_0041c2a0` | `Car_Init()` | Полная инициализация структуры: сброс координат, здоровья, флагов. Установка заглушек в двери. |
| `FUN_0041d5b0` | `Car_Move()` | **Ядро физики.** Применяет силу двигателя, трение, поворот колес. Обновляет `Velocity` и `Position`. |
| `FUN_0041e120` | `Car_CheckCollision()` | Проверка пересечения `BoundingBox` с картой и другими объектами. Вызывает обработчики ударов. |
| `FUN_0041e890` | `Car_CollisionOnCar()` | Обработка столкновения двух машин. Расчет импульса, нанесение урона (`Health`), отскок. |
| `FUN_0041f6c0` | `CarDoor_SetOpenState()` | Логика открытия/закрытия двери. Меняет `doorState` и запускает анимацию кадров. |
| `FUN_00420100` | `Car_UpdateEngine()` | Симмуляция ДВС. Рост оборотов, нагрев, проверка на перегрев или глохновение. |
| `FUN_00420550` | `Car_Explode()` | Смерть машины. Создание частиц огня, нанесение урона вокруг, удаление из пула. |

## 2. Структура `Ped` (Пешеход)
**Файл:** `include/ped.h`  
**Размер:** ~664 байта  
**Описание:** Пешеход, бандит, полицейский или игрок. Содержит ИИ, инвентарь, задачи и связи.

### Поля структуры

| Смещение | Старое имя (IDA) | Новое имя (Проект) | Тип | Описание |
|:---|:---|:---|:---|:---|
| **0x00** | `field_0x0` | `ObjectiveQueue[50]` | `Task[50]` | Очередь задач ИИ (пойти туда, убить этого, сесть в машину). |
| **0x190** | `field_0x190` | `QueueCounter` | `u8` | Индекс текущей выполняемой задачи в очереди. |
| **0x194** | `field_0x194` | `LinkedPed` | `Ped*` | Указатель на лидера банды или текущую цель атаки. |
| **0x198** | `field_0x198` | `CurrentCar` | `Car*` | Машина, в которой пешеход находится сейчас (как пассажир/водитель). |
| **0x19C** | `field_0x19c` | `TargetCar` | `Car*` | Машина, в которую пешеход пытается сесть прямо сейчас. |
| **0x1A0** | `field_0x1a0` | `Position` | `Vector3D` | Координаты в мире. Z используется для прыжков или лестниц. |
| **0x1AC** | `field_0x1ac` | `Heading` | `f32` | Направление взгляда и движения (угол). |
| **0x1B0** | `field_0x1b0` | `Health` | `u16` | Здоровье. 0 = смерть (запуск анимации ragdoll/tруп). |
| **0x1B2** | `field_0x1b2` | `Armor` | `u16` | Уровень брони (поглощает урон до здоровья). |
| **0x1B4** | `field_0x1b4` | `GangID` | `u8` | ID банды. Определяет цвет одежды и отношение к другим бандам. |
| **0x1B5** | `field_0x1b5` | `StateFlags` | `u8` | Флаги: 0x1-бежит, 0x2-стреляет, 0x4-руки вверх, 0x8-невидим. |
| **0x1B6** | `field_0x1b6` | `WeaponType` | `u8` | Тип оружия, которое сейчас в руках (для анимации). |
| **0x1B8** | `field_0x1b8` | `Inventory` | `WeaponSlot[4]` | Слоты экипировки: Кулак, Пистолет, Дробовик, Огнемет. |
| **0x1D0** | `field_0x1d0` | `SpritePtr` | `void*` | Указатель на структуру спрайта текущей анимации. |
| **0x1D4** | `field_0x1d4` | `AnimFrame` | `u16` | Номер текущего кадра анимации ходьбы/бега/стрельбы. |
| **0x1D6** | `field_0x1d6` | `AnimState` | `u8` | Состояние анимации (IDLE, WALK, RUN, DIE, SHOT). |
| **0x1D8** | `field_0x1d8` | `WantedLevel` | `u8` | Личный уровень розыска (актуально для игрока). |
| **0x1DC** | `field_0x1dc` | `Money` | `u32` | Количество денег при себе. Выпадают при смерти. |
| **0x1E0** | `field_0x1e0` | `AIContext` | `PedAI` | Контекст искусственного интеллекта (цели, восприятие, личность). |

## 3. Система `WantedSystem` (Розыск)

### Поля структуры

| Смещение | Старое имя (IDA) | Новое имя (Проект) | Тип | Описание |
|:---|:---|:---|:---|:---|
| **0x00** | `field_0x0` | `PlayerStars` | `u8` | Текущий уровень розыска (0 - 10 звезд). |
| **0x01** | `field_0x1` | `CooldownTimer` | `u16` | Таймер обратного отсчета. Если не преступничать, звезды падают. |
| **0x04** | `field_0x4` | `LastCrimePos` | `Vector2D` | Координаты последнего преступления. Полиция ищет здесь. |
| **0x0C** | `field_0xc` | `SearchRadius` | `f32` | Радиус, в котором полиция активно ищет игрока. |
| **0x10** | `field_0x10` | `CopSpawnRate` | `u8` | Частота появления подкреплений (зависит от звезд). |

### Функции системы

| Старое имя (IDA) | Новое имя (Проект) | Описание |
|:---|:---|:---|
| `FUN_00450100` | `Wanted_AddStars()` | Добавление звезд за убийство, кражу, таран копов. |
| `FUN_00450300` | `Wanted_RemoveStars()` | Снижение уровня (через время или аптечку/чит). |
| `FUN_00450500` | `Wanted_Update()` | Ежекадровый апдейт: уменьшение таймера, спавн патрулей. |
| `FUN_00450800` | `Wanted_SpawnReinforcement()` | Создание машины полиции в зоне видимости, но не на экране. |
