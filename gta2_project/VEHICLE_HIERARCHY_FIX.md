# Исправление иерархии транспортных средств GTA2

## Дата: 2024-04-20

## Проблема
Была перепутана иерархия между S81 (PublicTransport) и S82 (BaseCar).

## Правильная иерархия

```
BaseCar (S82, 0x68 байт) - БАЗОВЫЙ КЛАСС
└── Общая логика физики автомобилей
    - Позиция, скорость, ускорение
    - Коллизии, трение, гравитация
    - Базовые состояния (движение, остановка)

PublicTransport (S81, 0x7C байт) - НАСЛЕДНИК BaseCar
└── Логика общественного транспорта
    - Маршруты (RouteNodesPtr)
    - Остановки, таймеры остановок
    - Пассажиры, посадка/высадка
    - Специфичные флаги маршрута
```

## Изменения в файлах

### 1. include/entities/vehicle_types.h (СОЗДАН)
Новый файл с правильными определениями структур:
- `struct S82_BaseCar` - базовый класс (0x68 байт)
- `struct S81_PublicTransport` - наследник (0x7C байт)
- Вспомогательные enum: `VehicleType`, `PublicTransportState`, `RouteFlags`

### 2. include/structures_s.h (ИСПРАВЛЕН)
- Добавлен typedef для S81 (строка 65)
- Добавлено определение struct S81 после struct S82 (строки 27472-27520)
- Исправлен порядок: S81 теперь определяется как наследник S82

| Смещение | Размер | Поле | Описание |
|----------|--------|------|----------|
| 0x00 | 4 | PositionX | Координата X |
| 0x04 | 4 | PositionY | Координата Y |
| 0x08 | 4 | PositionZ | Координата Z |
| 0x0C | 4 | Heading | Угол поворота |
| 0x10 | 4 | Speed | Скорость |
| 0x14 | 4 | VehicleType | Тип ТС |
| 0x18 | 4 | Health | Здоровье |
| 0x1C | 4 | MaxHealth | Макс. здоровье |
| 0x20 | 4 | Flags | Флаги |
| 0x24 | 4 | State | Состояние |
| 0x28 | 4 | Timer | Таймер |
| 0x2C | 2 | PassengerCount | Кол-во пассажиров |
| 0x2E | 1 | TrafficLightState | Светофор |
| 0x2F | 1 | DoorState | Двери |
| 0x30 | 1 | RouteIndex | Индекс маршрута |
| 0x31 | 1 | StopTimer | Таймер остановки |
| 0x32 | 1 | RouteFlags | Флаги маршрута |
| 0x33 | 1 | Reserved | Выравнивание |
| 0x34 | 4 | RouteNodesPtr | Узлы маршрута |
| 0x38-0x67 | ... | ... | Дополнительные поля физики |

## Структура S81 (PublicTransport) - 0x7C байт

Наследует все поля S82 (0x00-0x67), плюс:

| Смещение | Размер | Поле | Описание |
|----------|--------|------|----------|
| 0x68 | 4 | field34_0x68 | Данные PT |
| 0x6C | 4 | field35_0x6c | Данные PT |
| 0x70 | 4 | field36_0x70 | Данные PT |
| 0x74 | 4 | field37_0x74 | Данные PT |
| 0x78 | 4 | field38_0x78 | Данные PT |

## Ключевое поле: RouteNodesPtr

**Важно:** Поле `RouteNodesPtr` находится по смещению 0x34 в обеих структурах, 
так как S81 наследует S82. Это указатель на массив узлов маршрута для 
общественного транспорта.

## Функции PublicTransport (S81)

Следующие функции работают с структурой S81:

- `S81_FUN_0045dd20` - Инициализация/сброс
- `S81_FUN_004af420` - Получение данных
- `S81_FUN_004af680` - Обработка автомобиля
- `S81_FUN_004af700` - Проверка автомобиля
- `S81_FUN_004afe20` - Обновление состояния
- `S81_FUN_004b08a0` - Работа с маршрутом
- `S81_FUN_004b0d70` - Обработка события
- `S81_FUN_004b1560` - Таймер/счётчик
- `S81_FUN_004b1b40` - Поиск автомобиля
- `IsThisBus` - Проверка типа ТС
- `HasReachedBusSkipLimit` - Проверка лимита остановок

## Примечания

- S82 (BaseCar) используется как базовый класс для всех транспортных средств
- S81 (PublicTransport) специализирован для автобусов, такси, поездов
- Car (структура в entities/car.h) также может наследовать BaseCar
