# Пометки для будущих чатов (GTA 2 Reverse Engineering)

## Ключевые принципы работы:
1. **Строго по дампу**: Не придумывать функции, поля или логику. Только то, что видно в ассемблерном листинге (gta2.exe.asm) и MAP-файле.
2. **Структуры**: Каждая структура — отдельный `.h` файл + пара `.cpp` (если есть методы). Размеры должны совпадать с оригиналом.
3. **Адреса и имена**: Сохранять оригинальные адреса функций и имена из MAP-файла.
4. **Папки проекта**: 
   - `gta2_project_c` — C-код
   - `gta2_project_cpp` — C++ код
   - `gta2.exe/ida_pro/` — исходные дампы (asm, map)
5. **Сборка**: Всё должно собираться в рабочий `.exe`.

## Текущий статус:
- Проведена ревизия: удалены несуществующие функции.
- Оставлены только конструкторы/деструкторы и поля, подтвержденные листингом.
- Возможны расхождения между локальными файлами и Git из-за сбоя связи.

## Что проверять при старте нового чата:
1. Прочитать "Работа.md" и "требование.md".
2. Сверить файлы в `include/entities/*.h` и `src/*/*.c(pp)` на наличие выдуманных функций.
3. Убедиться, что размеры структур соответствуют бинарнику.

## Контакты и ресурсы:
- Основной источник: дизассемблерный листинг IDA Pro.
- Критично: не допускать "творчества" в восстановлении кода.
